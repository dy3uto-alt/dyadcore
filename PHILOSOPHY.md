# dyadcore 记忆哲学

## 主流 Benchmark 范式的局限

主流记忆 benchmark（LongMemEval 等）的基本假设：系统存入信息 → 之后查询 → 看能不能召回。记忆被当作数据库条目来评测。

dyadcore 走的是另一条路。

## 1. 记忆是痕迹，不是数据

记忆不是"存入后被检索的数据"，而是用户和 Agent 在关系场里共同留下的**痕迹**。一条记忆的价值不在于它是否被正确召回，而在于它和别的痕迹之间构成了什么样的关系网。

Benchmark 测"命中率"，dyadcore 关心的是"留了什么轨迹，它们之间是怎么相互作用的"。

## 2. 图结构优先于语义相似度

相关性 ≈ 图拓扑信号，而非向量空间距离。

排序由 contradicted 惩罚、echoed 确认、field 场强、belief 证据加成这些**关系指标**驱动。FTS5 trigram 只负责初筛，真正的排序重任由图结构完成。

dyadcore 在传统的 MRR / NDCG 指标上可能不如专用检索系统——这不是 bug，这是设计取舍。它衡量的是"关系场里的位置"，而不是"内容与查询的匹配度"。

## 3. 记忆不随时间衰减

几乎所有记忆 benchmark 都隐含地偏袒"最近写入的更相关"。dyadcore 明确拒绝了这个假设：**旧记忆是未点燃的信号，价值取决于图结构而非年龄**。

一条三周前被锚定的矛盾关系，在排序中应该压倒三天前的弱相关消息。如果 benchmark 场景本身是时间相关的（比如"用户上周换了偏好"），dyadcore 依赖 contradicted 边而非时间衰减来进化——这构成一种独特的评测维度。

## 4. Beliefs — 涌现的认知状态

主流 benchmark：写了"用户用 React"，之后问"用户用什么框架"，看能不能答对。

dyadcore：Beliefs 是从 contradicted 边的拓扑中**自动合成**的。它们不是被写入的，而是从关系网中涌现的。置信度也是纯结构的：

```
confidence = f(证据量, echoed 信号, 源多样性, contradicted 惩罚)
```

不依赖外部权威。这个维度的评测不在任何现有 benchmark 里——你需要测的不是"记住了吗"，而是"图里长出来的信念正确吗"。

## 5. 去 Agent 中心化

主流记忆评测把 Agent 当成记忆主体，用户信息是"被记住的对象"。Dual Mirror 是双向的——用户和 Agent 各留痕迹，关系边是双向的。

Agent 对自己的反思、观察同样参与图结构排序。Agent 的自我认知也会影响后续检索结果。

## 总结

| 维度 | 主流 Benchmark | dyadcore |
|------|---------------|----------|
| 记忆本质 | 存入→检索的数据 | 关系中留下的痕迹 |
| 排序逻辑 | 语义相似度 + 时间衰减 | 图拓扑信号（关系边） |
| 时间观 | 越近越相关 | 不衰减，图结构决定价值 |
| Beliefs | 无此概念 | 涌现的认知状态，纯结构置信度 |
| 主体地位 | Agent 中心 | 双向对等 Dual Mirror |
| 评测适合度 | MRR, NDCG, Hit Rate | 尚无适配的评测框架 |

现有 benchmark 能测 dyadcore 的**下限**（基本检索能力），但测不了它的**上限**（图拓扑涌现的认知质量）。后者可能需要一套全新的评测范式。
