dyadcore

AI Agent 本地 Peer 记忆系统。零依赖，SQLite 原生。
用户与 Agent 是磁场的两极，记忆是发生在场中的痕迹。 Local peer memory for AI agents. Zero dependencies. SQLite-native. User and agent are two poles of a magnetic field — memory is the trace left in the field.

Architecture

User Mirror

source = "user"

Records user behavior and expressions — every interaction is a trace.

Agent Mirror

source = "agent"

Records agent observations and reasoning — symmetric peer, no special category.

triggered echoed contradicted related

Cross-trace relation edges form a dynamic semantic graph. Retrieval expands along these edges (1-hop) to discover neighbors that are relevant but not directly matched.

Quick Start

pip install dyadcore

from dyadcore import DyadCore
dc = DyadCore("agent_memory.db")
# Write traces uid = dc.write("用户偏好本地部署", source="user", field="技术选型") dc.anchor(uid)
# Recall with semantic search + graph expansion results = dc.recall("本地记忆方案", field_hint="技术选型")
dc.close()

Core API

Method
write(content, *, source, field)Write a trace, returns memory_id
write_batch(items)Bulk write in single transaction (5–10x faster)
recall(query, *, field_hint, limit)Semantic recall with FTS5 + LIKE + graph expansion
anchor(mid) / unanchor(mid)Mark / unmark a reference node
archive(mid) / unarchive(mid)Archive / restore a memory
add_relation(src, tgt, type, strength)Add a relation edge between traces
get_relation_graph(mid, depth)Subgraph via recursive CTE
get_contradicted_map(ids)Old→new mapping for evolution chains
add_belief(subject, statement, field)Declare a belief (manual or auto-synthesized)
confirm_belief(bid) / contest_belief(bid)Confirm or challenge a belief with new evidence
stats() / field_snapshot(field)Database stats and field-level diagnostics
check_silence(days)Find dormant memories

Constraints

Single-file SQLite — no backend service, no config files.

Zero external dependencies — no sqlite-vec, no embedding model, no C extensions.

FTS5 trigram is the only retrieval engine, built into SQLite.

Python ≥ 3.10. Apache 2.0.
