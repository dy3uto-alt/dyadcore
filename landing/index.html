dyadcore

AI Agent 本地 Peer 记忆系统。零依赖，SQLite 原生。 用户与 Agent 是磁场的两极，记忆是发生在场中的痕迹。 Local peer memory for AI agents. Zero dependencies. SQLite-native. User and agent are two poles of a magnetic field — memory is the trace left in the field.

GitHub · PyPI

记忆不是存进去再查出来的数据。用户说的、Agent 回应的，都在一个场里留下痕迹。记忆的价值不取决于被命中多少次，取决于它和别的痕迹构成了什么样的关系网。图结构里的确认、矛盾、关联，这些信号比语义相似度更值得信。

Memory isn't data you store and query. What the user says, what the agent replies — all of it leaves traces in a shared field. A memory's value isn't how many times it gets hit. It's the web of relations it forms — confirmations, contradictions, associations. Those structural signals matter more than semantic similarity.

Beliefs 是从关系图的矛盾里长出来的，不是谁写进去的。置信度也不靠外部打分，就看证据多不多、有没有被回声确认过、是不是用户和 Agent 双源都在说——纯结构置信度。旧记忆也不做时间衰减，一条三周前被矛盾掉的痕迹，排序里就该盖过三天前的弱相干。记忆的价值在结构里，不在时间线上。

Beliefs emerge from contradictions in the graph — they aren't authored. Confidence is purely structural: how much evidence, whether there's echoed confirmation, whether both user and agent are speaking. No external scoring. Old memories don't decay — a contradiction from three weeks ago should outrank a weak match from three days ago. Value lives in structure, not on a timeline.

大多数记忆方案把 Agent 当记忆的中心，用户只是被记住的对象。但现实不是这样的。用户和 Agent 是磁场里对等的两极，痕迹是双向留下的。没有谁观察谁——同一个场里反复触动、确认、修正。把这个叫去 Agent 中心化。这个立场可能比单项技术选择更能影响记忆系统长期长成什么样。

Most memory systems put the agent at the center — the user is just the one being remembered. That's not how real conversation works. User and agent are two equal poles in a magnetic field, leaving traces in both directions. No observer and observed — just repeated triggering, echoing, and revision happening in the same field. This is de-agent-centric design. It might shape long-term memory behavior more than any single technical choice.

现有的记忆 benchmark——MRR、NDCG、Hit Rate——测的是检索系统的命中率。dyadcore 在这些指标上不一定会赢，因为它的排序权重不在语义匹配上，在图结构信号上。测"记住了吗"，图结构排序给的是下限。测"图里长出来的认知对吗"，现有 benchmark 还没有这道题。一个靠 contradicted 边进化、不做时间衰减、从图拓扑里涌现信念的系统，需要一套自己的评测维度。

Existing memory benchmarks — MRR, NDCG, Hit Rate — measure retrieval hit rates. dyadcore won't necessarily win on these. Its ranking weights aren't on semantic matching but on structural graph signals. That's not something to hide from. If you're asking "did it remember," graph-based ranking gives you a floor. If you're asking "are the beliefs that emerged from the graph correct," no existing benchmark has that question yet. A system that evolves through contradicted edges, refuses time decay, and lets beliefs surface from topology — it needs a new evaluation framework. That might be the most important thing to build next.
Architecture

User Mirror

source = "user"

Records user behavior and expressions — every interaction is a trace.

Agent Mirror

source = "agent"

Records agent observations and reasoning — symmetric peer, no special category.

triggered echoed contradicted related

Cross-trace relation edges form a dynamic semantic graph. Retrieval expands along these edges (1-hop) to discover neighbors that are relevant but not directly matched.

Quick Start

pip install dyadcore-mem
# or: git clone https://github.com/dy3uto-alt/dyadcore && cp dyadcore/dyadcore.py dyadcore/hermes_bridge.py ./

from dyadcore import DyadCore
dc = DyadCore("agent_memory.db")
# Write traces uid = dc.write("用户偏好本地部署", source="user", field="技术选型") dc.anchor(uid)
# Recall results = dc.recall("本地记忆方案", field_hint="技术选型")
dc.close()

Core API

  • write(content, *, source, field)Write a trace, returns memory_id
  • write_batch(items)Bulk write in single transaction
  • recall(query, *, field_hint, limit)FTS5 + LIKE + graph expansion
  • anchor(mid) / unanchor(mid)Mark / unmark reference node
  • archive(mid) / unarchive(mid)Archive / restore a memory
  • add_relation(src, tgt, type)Add a relation edge
  • get_relation_graph(mid, depth)Subgraph via recursive CTE
  • get_contradicted_map(ids)Old→new evolution chain mapping
  • add_belief(subject, statement, field)Declare a belief
  • confirm_belief(bid)Confirm or challenge a belief
  • stats() / field_snapshot(field)Stats and diagnostics
  • check_silence(days)Find dormant memories
